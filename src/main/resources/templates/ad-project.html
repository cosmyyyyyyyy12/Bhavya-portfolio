Project Write-Up

Asset Inventory Simulator

Java + XML Data Practice

Overview

This project is a beginner-friendly simulation of an internal asset inventory tool. It reads structured XML data and presents it through a portfolio-style interface, which helped me practice how backend data can be organized and displayed in a useful way.

What The Project Does

  • Stores sample computer and directory data in XML files.
  • Simulates command-style lookups for machines, groups, and inventory records.
  • Uses Java controllers to connect routes with frontend templates.
  • Turns raw backend data into a more readable project demo.

What I Practiced

  • Working with structured data files and understanding how they can model real systems.
  • Keeping backend and frontend responsibilities separate inside a Spring Boot project.
  • Designing a project that feels interactive without becoming too complex.
  • Explaining technical ideas clearly through a portfolio write-up.

Why It Matters

As a beginner, I wanted to build something that was more than a static page. This project gave me a way to combine backend logic, structured data, and presentation. It also helped me think about how admin tools and dashboards can be simplified for users.

Interactive Demo

The page includes a small PowerShell-style console so visitors can explore the mock environment. That part of the project helped me practice routing, frontend interaction, and simple backend responses all together.

  • Get-ADUser
  • Get-ADComputer
  • Get-ADGroup
  • Get-ADOrganizationalUnit
  • Get-GPO
  • Get-ADDomainController
  • Get-ADGroupMember -Identity "Group"
  • Get-Content -Path "AssetInventory \ ComputerName"

Next Steps

I would like to keep improving this project by adding search filters, cleaner data views, and a stronger service layer for handling inventory records. Long term, I would also like to swap some of the XML-backed content into a database-backed version.
